================================================================================
                              WHAT IS SOULSYNC AI?
================================================================================

SoulSync AI is a personal AI companion system that:

  - Remembers EVERY conversation you have with it -- forever
  - Understands your emotions, habits, goals, and personality
  - Stores structured facts about you (name, age, job, goals, family, opinions)
  - Automatically detects and creates tasks from your messages
  - Gives smart personalized suggestions based on your life patterns
  - Talks to you through TEXT or full VOICE mode (like Alexa)
  - Responds in a natural Indian female voice (Microsoft Neerja Neural)
  - Supports English, Hindi (Devanagari), and Hinglish (Roman script)
  - Classifies every message into 1 of 32 life collection types
  - Tracks your mood with 74 distinct mood states
  - Supports multiple users -- each person has their own private memory
  - Runs on MongoDB (cloud) + FAISS (local vectors) + Redis (cache)

Unlike regular chatbots that forget everything after each session,
SoulSync AI builds a permanent, growing model of your life.

================================================================================
                    COMPLETE WORKFLOW PIPELINE -- STEP BY STEP
================================================================================

This is the exact journey of every message through SoulSync AI.

--------------------------------------------------------------------------------
STEP 1: INPUT (Text or Voice)
--------------------------------------------------------------------------------

  TEXT MODE:
    User types in the chat window.
    Example: "I had a tough day. Stressed about work, skipped gym."

  VOICE MODE (Alexa-style):
    User clicks "Voice" button in header.
    Full-screen dark interface with animated sphere appears.
    User taps sphere -- it starts listening via Web Speech API (Chrome).
    User speaks naturally. No buttons between turns.
    After AI responds (via Neerja TTS), mic reopens automatically.

--------------------------------------------------------------------------------
STEP 2: LANGUAGE DETECTION
--------------------------------------------------------------------------------

  SoulSync detects which language the user is writing in:

  ENGLISH:
    "I am feeling stressed today"
    -> Detected: English
    -> AI responds in English

  HINDI (Devanagari script):
    "Aaj mujhe bahut thaka hua feel ho raha hai"
    -> Detected: Hinglish
    -> AI responds in Hinglish

  HINGLISH (Hindi words in Roman script):
    "Mera naam Rohit hai aur main Mumbai mein rehta hoon"
    -> Detected: Hinglish
    -> AI responds in Hinglish

  Detection is instant (pattern-based, no API call needed).
  Language instruction is injected into the AI system prompt.

--------------------------------------------------------------------------------
STEP 3: INTENT DETECTION (5 intents)
--------------------------------------------------------------------------------

  Every message is classified into one of 5 intents:

  1. PERSONAL_INFO_STORE
     Triggered by: "My name is...", "I am 24 years old", "I live in..."
     Action: Extracts key-value pair, saves to MongoDB memories collection
     Examples:
       "My name is Rohit"          -> key=name, value=Rohit
       "My goal is to be a doctor" -> key=goal, value=be a doctor
       "I work as an engineer"     -> key=job, value=engineer
       "Mera naam Rohit hai"       -> key=name, value=Rohit (Hinglish)
     Supported keys: name, age, job, workplace, goal, dream, aim, hobby,
                     interest, location, email, phone, partner, family,
                     education, degree, favorite_*, opinion_*

  2. PERSONAL_INFO_QUERY
     Triggered by: "What is my name?", "What is my goal?", "Tell me about me"
     Action: Looks up MongoDB memories, returns direct answer (no AI needed)
     Special: "What was the first thing I shared?" -> returns oldest message
     Hindi: "Mera naam kya hai?", "Mera goal kya hai?"

  3. TASK_COMMAND
     Triggered by: "Remind me to...", "I need to finish...", "Schedule..."
     Action: Detects task, creates it in MongoDB tasks collection
     Examples:
       "Remind me to call doctor tomorrow" -> Task: Call doctor | Due: tomorrow
       "I need to submit report by Friday" -> Task: Submit report | Due: Friday
       "Mujhe kal gym jaana hai"           -> Task: Gym | Due: tomorrow

  4. TASK_MANAGE
     Triggered by: "I finished the gym task", "Delete the grocery task"
     Action: Completes or deletes existing task
     Operations: complete, delete, priority_high, priority_low, priority_medium

  5. NORMAL_CHAT
     Everything else -> goes to full RAG pipeline with AI

--------------------------------------------------------------------------------
STEP 4: COLLECTION CLASSIFICATION (32 collections)
--------------------------------------------------------------------------------

  Every user message is classified into one of 32 life collection types.
  This determines how the memory is stored and retrieved.

  IDENTITY & SELF:
    identity_fact      -- name, age, location, job facts
    personality_trait  -- "I am an introvert", "I tend to overthink"
    belief_value       -- religion, philosophy, core values
    self_reflection    -- realizations, growth, lessons learned

  EMOTIONS & MENTAL HEALTH:
    emotion_log        -- daily feelings, mood entries
    mental_health      -- anxiety, depression, therapy, burnout
    gratitude          -- thankfulness, appreciation
    fear_worry         -- fears, anxieties, worries

  RELATIONSHIPS & SOCIAL:
    relationship       -- family, friends, partner interactions
    conflict           -- fights, arguments, tensions
    social_event       -- parties, weddings, gatherings
    loss_grief         -- death, breakups, endings

  LIFE EVENTS & EXPERIENCES:
    experience         -- travel, events, things that happened
    achievement        -- wins, promotions, completions, milestones
    failure_setback    -- failures, rejections, disappointments
    decision           -- choices made, dilemmas, crossroads
    milestone          -- birthdays, anniversaries, firsts
    surprise           -- unexpected events

  GOALS & FUTURE:
    goal               -- long-term goals, life goals
    future_plan        -- upcoming plans, intentions
    dream_aspiration   -- bucket list, wishes, fantasies
    habit              -- routines, streaks, daily practices

  HEALTH & BODY:
    health             -- workouts, illness, diet, sleep, medical

  WORK & CAREER:
    work_career        -- job events, projects, work stress

  FINANCIAL:
    financial          -- money, salary, spending, savings, debt

  LEARNING & GROWTH:
    learning           -- books, courses, skills, discoveries
    creative_work      -- writing, art, music, coding projects

  OPINIONS & THOUGHTS:
    opinion            -- views, beliefs, takes on topics
    life_lesson        -- wisdom gained, advice to self

  SPECIAL MOMENTS:
    humor_fun          -- funny moments, jokes, lighthearted memories
    personal_fact      -- structured personal data

  GENERAL:
    conversation       -- fallback for everything else

  Importance scores (1-10) are assigned per collection:
    achievement, milestone, loss_grief = 9 (highest)
    failure_setback, experience, decision = 8
    conflict, self_reflection, goal, life_lesson = 7
    work_career, social_event, relationship, creative_work = 6
    emotion_log, health, habit, financial, learning = 5
    dream_aspiration, opinion, humor_fun = 4
    conversation = 3 (lowest)

--------------------------------------------------------------------------------
STEP 5: MEMORY STORAGE (MongoDB)
--------------------------------------------------------------------------------

  Every message is saved to MongoDB in two places:

  A) messages collection (raw conversation):
     {
       message_id      : UUID
       conversation_id : UUID (one per month per user)
       user_id         : "rohit_seed"
       role            : "user" or "assistant"
       content         : "I had a tough day..."
       importance_score: 7 (auto-calculated)
       emotion         : "stressed"
       intent          : "normal_chat"
       created_at      : 2025-06-15T14:30:00
     }

  B) memories collection (personal facts only, for personal_info_store):
     {
       memory_id   : UUID
       user_id     : "rohit_seed"
       key         : "goal"
       value       : "Become a senior engineer"
       context     : "goal"
       source_text : "My goal is to become a senior engineer"
       event_date  : null
       created_at  : 2025-01-01
       updated_at  : 2025-06-15
     }

  C) activities collection (structured extraction):
     {
       activity_id  : UUID
       user_id      : "rohit_seed"
       raw_text     : "I felt tired and skipped gym"
       emotion      : "tired"
       activity     : "gym"
       status       : "missed"
       productivity : "low"
       summary      : "User felt tired and missed gym"
       created_at   : 2025-06-15
     }

  D) FAISS vector index (semantic search):
     Message text -> 384-dim embedding -> stored in user's FAISS index
     File: data/vectors/{user_id}.index
     File: data/vectors/{user_id}.meta.json

--------------------------------------------------------------------------------
STEP 6: MOOD AUTO-LOGGING
--------------------------------------------------------------------------------

  After every message, SoulSync auto-detects the user's mood.
  74 distinct mood states are supported:

  POSITIVE (score 7-10):
    ecstatic(10), elated(10), euphoric(10), excited(9), motivated(9),
    inspired(9), energetic(9), happy(8), joyful(8), proud(8),
    confident(8), grateful(8), optimistic(8), thrilled(8),
    content(7), peaceful(7), calm(7), relaxed(7), focused(7),
    satisfied(7), hopeful(7), refreshed(7)

  NEUTRAL (score 4-6):
    neutral(5), okay(5), fine(5), alright(5), normal(5),
    meh(4), indifferent(4), mixed(4), bored(4), restless(4),
    confused(4), uncertain(4), lonely(4), nostalgic(4)

  NEGATIVE (score 1-3):
    tired(3), exhausted(3), drained(3), stressed(3), burnout(3),
    anxious(3), nervous(3), worried(3), overwhelmed(3),
    frustrated(3), irritated(3), disappointed(3), sad(3),
    empty(3), numb(3), hurt(3), lost(3),
    angry(2), furious(2), devastated(2), heartbroken(2),
    depressed(2), hopeless(2), miserable(2), scared(2),
    terrified(2), panicked(2), broken(2),
    suicidal(1), destroyed(1)

  Mood is logged to mood_logs collection with:
    - mood name, score, day of week, hour of day, note, source

--------------------------------------------------------------------------------
STEP 7: RAG RETRIEVAL (for normal_chat intent)
--------------------------------------------------------------------------------

  Before generating an AI response, SoulSync retrieves relevant context:

  LAYER 1 -- Personal Facts (MongoDB memories collection):
    Fetches all stored facts: name, age, job, goal, hobby, location, etc.
    Formats as: "Known facts about this user:
- Name: Rohit
- Goal: ..."
    Cached in Redis for 5 minutes (key: facts:{user_id})

  LAYER 2 -- Semantic Memory Search (FAISS):
    Embeds the current message using all-MiniLM-L6-v2 (384 dimensions)
    Searches user's FAISS index for top-5 most similar past messages
    Distance threshold: 1.2 (filters out irrelevant memories)
    Returns: list of {text, score} dicts

  LAYER 3 -- Keyword Fallback (MongoDB):
    If FAISS returns nothing, extracts keywords from message
    Searches MongoDB messages collection using regex
    Returns up to 3 matching past messages

  LAYER 4 -- Chat History (last 5 turns):
    Fetches last 5 user+assistant message pairs from current conversation
    Provides short-term context for the AI

  All layers combined into one context string injected into AI prompt.

--------------------------------------------------------------------------------
STEP 8: AI RESPONSE GENERATION (Groq API)
--------------------------------------------------------------------------------

  SoulSync uses Groq API with llama-3.3-70b-versatile model.

  System prompt includes:
    - SoulSync AI persona (built April 23, 2026 by Surjeet)
    - Today's date
    - Language instruction (respond in user's language)
    - User's personal facts (from Layer 1)

  Message history includes:
    - Last 5 conversation turns (short-term memory)
    - Relevant past memories (from FAISS/keyword search)
    - Current user message

  Direct-answer shortcuts (no API call needed):
    "Who are you?"        -> hardcoded SoulSync introduction
    "When were you built?"-> "April 23, 2026"
    "How do you feel?"    -> warm personal response
    "What is today?"      -> current date

  Response time: 0.3 - 0.7 seconds (Groq is very fast)

--------------------------------------------------------------------------------
STEP 9: TASK AUTO-DETECTION
--------------------------------------------------------------------------------

  If intent = task_command, SoulSync detects tasks from the message:

  Detection patterns include:
    - "Remind me to [action]"
    - "I need to [action] by [date]"
    - "I have to [action] [time]"
    - "Don't let me forget to [action]"
    - "Schedule [action] on [date]"
    - "Mujhe [action] karna hai" (Hinglish)
    - "Yaad dilao mujhe ki [action]" (Hinglish)

  Extracted fields:
    - title: the task description
    - due_date: today/tomorrow/Friday/next week/etc.
    - priority: high/medium/low (inferred from urgency words)
    - source: "auto" (vs "manual" for user-created tasks)

  Saved to MongoDB tasks collection.
  Shown in Tasks panel in the sidebar.

--------------------------------------------------------------------------------
STEP 10: RESPONSE CACHING (Redis)
--------------------------------------------------------------------------------

  Redis caches responses to avoid redundant API calls:

  Cache keys:
    chat:{user_id}:{md5_hash_of_message}  -- AI responses (TTL: 10 min)
    facts:{user_id}                        -- personal facts (TTL: 5 min)
    recent_chats:{user_id}                 -- conversation list (TTL: 5 min)
    session:{user_id}                      -- user session (TTL: 24 hours)

  Cache is skipped for:
    - personal_info_query intent (always needs fresh DB lookup)
    - memory_query intent

  Cache hit rate: ~83%
  Redis is optional -- system works without it (graceful degradation)

--------------------------------------------------------------------------------
STEP 11: VOICE RESPONSE (Voice Mode only)
--------------------------------------------------------------------------------

  After AI generates text response:

  1. Text sent to POST /api/v1/voice/speak
  2. Backend runs edge-tts in dedicated ThreadPoolExecutor
     (required because pyttsx3/SAPI5 needs its own COM thread)
  3. edge-tts generates MP3 audio using Microsoft Neerja Neural
     Voice: en-IN-NeerjaNeural (Indian English female)
     Rate: 160 wpm, Volume: 0.97
  4. MP3 bytes returned to frontend
  5. Browser plays audio through speakers
  6. After audio ends, mic reopens automatically
  7. Sphere returns to green (listening state)

  Fallback: pyttsx3 -> Microsoft Zira (offline, if edge-tts fails)

  Sphere states:
    Green  (listening)  : mic open, waiting for user speech
    Amber  (thinking)   : sent to AI, waiting for response
    Violet (speaking)   : audio playing through speakers
    Gray   (paused)     : user paused the session
    Red    (expired)    : 10-minute session limit reached

--------------------------------------------------------------------------------
STEP 12: SUGGESTION ENGINE
--------------------------------------------------------------------------------

  Analyzes user's activity patterns using Pandas:

  Input: activities collection (emotion, activity, status, productivity)

  Detects patterns like:
    - "You skip gym when stressed (3 times this week)"
    - "Low productivity on Mondays"
    - "You feel happy after social activities"
    - "You've been feeling tired frequently"

  Generates personalized suggestions:
    - "Try a 10-minute walk instead on stressful days"
    - "Start Mondays with your easiest task to build momentum"
    - "Schedule more time with friends -- it boosts your mood!"

  Available via GET /api/v1/suggestions/{user_id}


================================================================================
                         DATABASE ARCHITECTURE
================================================================================

SoulSync AI v3.0 uses a MongoDB-only architecture (PostgreSQL removed).

PRIMARY DATABASE: MongoDB Atlas (cloud)
  Collections:
    users              -- user profiles, auth, preferences
    conversations      -- one per month per user (12 per user)
    messages           -- all chat messages (user + assistant)
    memories           -- personal facts (name, goal, job, etc.)
    tasks              -- user tasks (pending/completed)
    activities         -- structured emotion/activity extraction
    mood_logs          -- weekly mood tracking (52 per user per year)
    memory_collections -- typed life events (32 collection types)
    monthly_summaries  -- monthly aggregated summaries
    life_timeline      -- chronological life events

VECTOR SEARCH: FAISS (local disk)
  Files: data/vectors/{user_id}.index + {user_id}.meta.json
  Model: all-MiniLM-L6-v2 (384 dimensions)
  Per-user indexes, stored on local disk

CACHE: Redis (optional, local)
  Used for: AI responses, personal facts, conversation lists, sessions
  Graceful degradation if Redis is unavailable

DISABLED (future): MySQL
  Reserved for: wallet, transactions, subscriptions, payments
  Re-enable: set ENABLE_PAYMENTS=true in .env + configure MySQL

MONGODB COLLECTION SCHEMAS:

  users:         user_id, name, email, password_hash (bcrypt),
                 profile {age, job, location}, preferences, created_at, updated_at

  conversations: conversation_id, user_id, title, message_count,
                 last_message_at, created_at, updated_at

  messages:      message_id, conversation_id, user_id, role (user/assistant),
                 content, importance_score (1-15), emotion, intent, created_at

  memories:      memory_id, user_id, key, value, context, source_text,
                 event_date, created_at, updated_at

  tasks:         task_id, user_id, title, due_date, priority (high/medium/low),
                 status (pending/completed), source (manual/auto),
                 created_at, updated_at, completed_at

  activities:    activity_id, user_id, raw_text, emotion, activity,
                 status, productivity, summary, created_at

  mood_logs:     log_id, user_id, mood, mood_score (1-10), note,
                 day_of_week, hour_of_day, source, created_at

INDEXES:
  users:     user_id (unique), email (unique), created_at
  messages:  message_id (unique), conversation_id, user_id,
             user_id+created_at (compound), content (text search)
  memories:  memory_id (unique), user_id, user_id+key (compound)
  tasks:     task_id (unique), user_id, user_id+status (compound)
  mood_logs: log_id (unique), user_id, user_id+created_at (compound)

                         TROUBLESHOOTING GUIDE

VOICE MODE NOT WORKING:
  - Use Chrome browser (Web Speech API only works in Chrome)
  - Allow microphone permissions
  - Check if HTTPS is enabled (required for some browsers)

TTS NOT WORKING:
  - Check internet connection (edge-tts requires online)
  - Fallback to pyttsx3 (Microsoft Zira) if edge-tts fails
  - Verify edge-tts is installed: pip show edge-tts

MONGODB CONNECTION ERRORS:
  - Check MONGODB_URL in .env file
  - Verify MongoDB Atlas cluster is running
  - Check network access whitelist in Atlas

GROQ API ERRORS:
  - Verify GROQ_API_KEY is valid
  - Check API rate limits (free tier: 30 requests/minute)
  - Ensure model name is correct: llama-3.3-70b-versatile

REDIS NOT CONNECTING:
  - Start Redis: redis-server
  - Check REDIS_URL in .env (default: redis://localhost:6379)
  - System works without Redis (graceful degradation)

SLOW RESPONSES:
  - Check Groq API status (may be rate limited)
  - Monitor MongoDB query performance
  - Consider enabling Redis caching if not active

FRONTEND BUILD ERRORS:
  - Delete node_modules and run: npm install
  - Clear Vite cache: rm -rf node_modules/.vite
  - Check Node.js version (requires 20+)
